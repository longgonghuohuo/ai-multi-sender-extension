## ⚠️ 重要说明

### iframe 的工作原理

由于浏览器的**同源策略**限制，iframe 无法被 content script 直接操作。

**因此工作方式是：**

1. **展示层**：iframe 显示三个 AI 网页（用于并排查看）
2. **操作层**：需要在独立标签页打开 AI（用于接收输入）
3. **发送机制**：点击发送时，插件查找所有已打开的 AI 标签页，通过 content script 填充问题

### 推荐使用方式

**方式一：纯 iframe 模式**
- 只在 iframe 中手动操作
- 适合查看和对比
- 无需额外标签页

**方式二：混合模式（推荐）✅**
```bash
1. 保持全屏页面打开（iframe 展示）
2. 同时打开三个独立标签页：
   - https://chatgpt.com
   - https://claude.ai
   - https://gemini.google.com
3. 在全屏页面输入问题，点击发送
4. 问题自动填充到独立标签页
5. 在 iframe 或标签页查看回答
```

## 🐛 故障排查

### 问题 1：iframe 显示空白

**解决方案**：
```bash
1. 点击顶部的"刷新"按钮
2. 检查网络连接
3. 手动在 iframe 中登录
```

### 问题 2：点击发送没反应

**原因**：可能没有打开独立的 AI 标签页

**解决方案**：
```bash
1. 打开三个独立标签页
2. 确保页面在对话页面
3. 查看控制台是否有 "content script loaded"
4. 重试发送
```

### 问题 3：如何快速访问全屏页面

**解决方案**：
```bash
# 方法一：点击图标
点击浏览器工具栏的插件图标

# 方法二：收藏书签
1. 打开全屏页面
2. 按 Ctrl+D 收藏
3. 下次直接点击书签打开

# 方法三：记住地址
chrome-extension://[插件ID]/app.html
```
